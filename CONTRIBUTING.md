# Contributing to Kynward Dev Toolkit

Thank you for your interest in contributing.

This repository is intentionally limited to open-source-safe developer tooling and documentation. It does not publish the full Kynward game or private game content.

## Good contribution areas

- Developer tools.
- Documentation checks.
- Validation helpers.
- Smoke-test workflows.
- Data inspection utilities.
- Small public examples.
- Issue templates and maintainer workflows.
- Godot prototype support scripts that do not depend on private content.

## Do not contribute

Do not add:

- private Kynward game content;
- unreleased lore or narrative canon;
- final art, logos, icons, music, sound, fonts, screenshots, or mockups;
- third-party assets without a compatible license;
- generated images intended to represent final game art;
- commercial design documents;
- secrets, tokens, credentials, local environment files, or deployment data;
- code copied from proprietary projects;
- large rewrites without prior discussion.

## Development principles

1. Keep developer tooling separate from player-facing game content.
2. Prefer small, reviewable changes.
3. Document validation rules and assumptions.
4. Keep examples minimal and public-safe.
5. Avoid hidden dependencies on private repositories.
6. Do not expose raw private paths, diagnostics, credentials, or machine-specific data.
7. Use English for documentation, comments, issue text, and pull request descriptions.

## Pull request checklist

Before opening a pull request, confirm:

- [ ] The change is inside the public toolkit scope.
- [ ] No private Kynward content is included.
- [ ] No third-party asset or copied code is included without a compatible license.
- [ ] Documentation is updated when behavior changes.
- [ ] Public examples remain small and synthetic.
- [ ] Comments and descriptions are written in English.
- [ ] The change can be understood without access to the private game repository.
