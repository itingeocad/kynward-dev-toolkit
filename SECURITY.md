# Security Policy

## Supported scope

Security reports are accepted for the public source code, developer tools, scripts, public examples, documentation tooling, and validation helpers in this repository.

This repository is not a production service and should not be treated as a deployed security boundary.

## Please do not report

- Issues in third-party services that are not controlled by this repository.
- Speculative reports without a reproduction path.
- Social engineering or phishing tests.
- Denial-of-service testing.
- Scans against systems that you do not own or have permission to test.
- Issues involving private assets, unreleased content, or confidential materials.

## Reporting

For now, open an issue only for non-sensitive security concerns.

For sensitive reports, contact the maintainer privately before publishing details. If GitHub private vulnerability reporting is enabled later, use that channel.

A good report should include:

- affected file, tool, or subsystem;
- steps to reproduce;
- expected behavior;
- actual behavior;
- impact;
- suggested fix, if available.

## Responsible disclosure

Please give the maintainer reasonable time to review and fix security issues before public disclosure.
