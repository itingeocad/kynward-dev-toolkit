# Public Roadmap

This roadmap describes the public toolkit direction only. It does not represent the full private Kynward game roadmap.

## Current status

The repository has a working public foundation: documentation boundaries, Apache 2.0 licensing, contribution rules, public-safe examples, dependency-free validation tools, GitHub Actions checks, and unit-style tests for the Markdown checkers.

Current open work is tracked in public issues. New contributions should stay small, synthetic, English-only, and independent of the private Kynward game repository.

## Application readiness

- [x] Keep the repository public.
- [x] Keep source code and documentation under Apache License 2.0.
- [x] Keep project name, lore, visual identity, concept art, music, screenshots, and other non-code materials outside the Apache 2.0 grant unless explicitly stated.
- [x] Keep public examples synthetic and independent of private game content.
- [x] Maintain GitHub Actions checks for public tooling.
- [x] Keep open issues for contributor-friendly next steps.
- [ ] Review and refine the Codex for Open Source application draft. See issue #10.

## Non-goals

- Publishing private Kynward game content.
- Publishing final game art or brand assets.
- Publishing unreleased mechanics or narrative canon.
- Recreating the full private repository in public.
